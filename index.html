1. Taller de Código: QDMesh

Interactúe con los parámetros a la derecha para ver cómo cambian el código Wolfram y los resultados de la simulación en tiempo real. Pase el mouse por el código para ver la documentación de las funciones.

Live Sandbox
simulation_script.nb
Needs["QDMesh`Physics`"];
Needs["QDMesh`Crypto`"];
distancia = 1000; (* Metros *)
viento = 5; (* m/s *)
nFotones = 50000;
atm = AtmosphericState[h, viento, distancia, 0.02];
qkd = RunProtocol["BB84", 0.05 + (atm["BeamWander_Var"] * 1000), nFotones];

TU FRAMEWORK
QD

QDMesh (Wolfram)

Especializado en física ambiental y modelado matemático simbólico.

  • Ventaja: Cálculo simbólico de turbulencia atmosférica.
  • Ventaja: Integración nativa con visualización de datos de Wolfram.
  • Mejor para: Diseño de enlaces físicos, Drones, Análisis de QBER por clima.

SeQUeNCe Python

Simulador de eventos discretos enfocado en hardware.

+ Gran detalle en memorias cuánticas y detectores.
- Curva de aprendizaje alta para configurar hardware.

QuNetSim Python

Enfoque en capas de red y protocolos de enrutamiento.

+ Fácil para probar algoritmos de routing.
- Física del canal (aire/fibra) muy simplificada.

PyFSO Python

Librería específica para óptica de espacio libre.

+ Bueno para cálculos de atenuación.
- No simula protocolos cuánticos (QKD) nativamente.

3. Integración en sus Proyectos

Redes de Drones Dinámicas

Referencia: "Dynamic Drone Mesh Networks..."

Use QDMesh`Physics` para modelar cómo el movimiento rápido de los drones altera el Beam Wander. QDMesh permite recalcular la matriz de atenuación en cada paso de tiempo `t` mucho más fácil que SeQUeNCe, que requiere redefinir la topología.

Comunicaciones Robóticas Seguras

Referencia: "Simulation of Secure Mobile Robot..."

Use QDMesh`Crypto` para simular el intercambio de claves (BB84) entre robots terrestres. A diferencia de QuNetSim, QDMesh le permite introducir modelos de ruido específicos del terreno (polvo, obstáculos) directamente en la ecuación de QBER.
